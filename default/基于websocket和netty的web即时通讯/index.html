基于websocket和netty的web即时通讯

  现在新的浏览器逐步都支持websocket了，而项目上也已经有基于netty
的及时通讯所以想结合两者来实现web即时通讯.
目前项目上是使用flash与netty通信的但是有跨域的问题和flash支持的问题
增加webscoket协议之后可以将多种框架柔和到一起，在浏览器客户端上判断
是否支持websocket如果支持直接使用websocket，如果不支持再使用flash。
之前的业务逻辑代码是不能变的，但是flash调用js方法接口写死的所以需要
整合两种通信方式提供统一的业务接口。还要实现服务器端安全认证，防止非
正常数据交互
